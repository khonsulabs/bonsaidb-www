BonsaiDb

A Rust database that grows with you.


BonsaiDb is currently pre-alpha. We are nearing the first alpha release. While there has been significant testing behind BonsaiDb already, all users are recommended to restrict BonsaiDb to experiments or ensure you test your backups.

Getting Started with Local-only Databases

Add BonsaiDb to your Cargo.toml:

[dependencies]
bonsaidb = { git = "https://github.com/khonsulabs/bonsaidb.git", branch = "main", features = ["local"] }

Local-only Examples

Each of these examples can be run within the BonsaiDb repository by executing cargo run --example EXAMPLE-NAME.

  • basic-local: Shows using one database with one collection.
  • basic-local-multidb: Shows using one collection with multiple databases.
  • view-examples: Shows basic Map/Reduce views usage for collections.
  • view-histogram: Demonstrates using a histogram as a value type within a Map/Reduce view, enabling automatic aggregation of histograms through queries.
  • pubsub: Demonstrates publishing and subscribing to events published to topics. While PubSub is arguably more useful in the client/server model, it works offline as well.
  • key-value-store: Shows basic usage of the atomic key-value store.

Getting Started with Client/Server Databases

Add BonsaiDb to your server's Cargo.toml:

[dependencies]
bonsaidb = { git = "https://github.com/khonsulabs/bonsaidb.git", branch = "main", features = ["server"] }

And to your client's Cargo.toml:

[dependencies]
bonsaidb = { git = "https://github.com/khonsulabs/bonsaidb.git", branch = "main", features = ["client"] }

Client/Server Examples

Each of these examples can be run within the BonsaiDb repository by executing cargo run --example EXAMPLE-NAME.

The examples for local-only databases are focused on teaching the basics on BonsaiDb and apply to all forms of accessing BonsaiDb, including Client/Server. If you're new to BonsaiDb, it might be best to start with the local-only examples while learning the fundamentals.

  • basic-server: Shows basic server setup and access. Example includes both QUIC and WebSocket based connections.
  • users: Shows basic user and permission group management.
  • custom-api: Shows how to extend the BonsaiDb network protocol with a custom API. See also the user's guide section covering this example.
  • cli: Shows how to use and extend the BonsaiDb command line interface.
  • axum: Shows how to expose an Axum webserver that supports upgrading BonsaiDb websockets on a specified route.
  • acme: Shows how to configure BonsaiDb to use LetsEncrypt to automatically issue TLS certificates.
